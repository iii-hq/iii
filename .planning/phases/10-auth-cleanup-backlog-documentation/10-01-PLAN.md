<objective>
Eliminate all `unsafe { std::env::set_var }` and `unsafe { std::env::remove_var }` calls from the auth module's test suite by migrating to the `temp-env` crate's closure-based API.

Purpose: The Rust 2024 edition made `std::env::set_var` unsafe due to thread-safety risks. The current auth tests use `unsafe` blocks paired with `#[serial]` for env var manipulation. `temp-env` provides a safe, mutex-synchronized, panic-cleanup-aware replacement that eliminates both the `unsafe` keyword and the need for `#[serial]` on these tests.

Output: All 18 auth tests pass with zero `unsafe` blocks and zero `#[serial]` attributes in the auth test module.
</objective>
