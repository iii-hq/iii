# Migrate from `modules` to `workers` config

This guide covers the **breaking change** that renamed the YAML configuration from `modules` / `class` to `workers` / `name`, and simplified all module and adapter identifiers.

## What changed

| Before | After |
|--------|-------|
| Top-level key `modules:` | Top-level key `workers:` |
| Entry field `class:` | Entry field `name:` |
| Adapter field `class:` | Adapter field `name:` |
| Rust-style paths (`modules::stream::StreamModule`) | Short identifiers (`iii-stream`) |
| New: n/a | `image:` field for Docker-based workers |

## Worker name mapping

| Old `class` value | New `name` value |
|---|---|
| `modules::worker::WorkerModule` | `iii-worker-manager` |
| `modules::stream::StreamModule` | `iii-stream` |
| `modules::state::StateModule` | `iii-state` |
| `modules::api::RestApiModule` | `iii-http` |
| `modules::pubsub::PubSubModule` | `iii-pubsub` |
| `modules::observability::OtelModule` | `iii-observability` |
| `modules::queue::QueueModule` | `iii-queue` |
| `modules::http_functions::HttpFunctionsModule` | `iii-http-functions` |
| `modules::cron::CronModule` | `iii-cron` |
| `modules::shell::ExecModule` | `iii-exec` |
| `modules::bridge_client::BridgeClientModule` | `iii-bridge` |

## Adapter name mapping

| Old `class` value | New `name` value |
|---|---|
| `modules::stream::adapters::KvStore` | `kv` |
| `modules::stream::adapters::RedisAdapter` | `redis` |
| `modules::stream::adapters::Bridge` | `bridge` |
| `modules::state::adapters::KvStore` | `kv` |
| `modules::state::adapters::RedisAdapter` | `redis` |
| `modules::state::adapters::Bridge` | `bridge` |
| `modules::queue::BuiltinQueueAdapter` | `builtin` |
| `modules::queue::RedisAdapter` | `redis` |
| `modules::queue::RabbitMQAdapter` | `rabbitmq` |
| `modules::queue::adapters::Bridge` | `bridge` |
| `modules::pubsub::LocalAdapter` | `local` |
| `modules::pubsub::RedisAdapter` | `redis` |
| `modules::cron::KvCronAdapter` | `kv` |
| `modules::cron::RedisCronAdapter` | `redis` |

## Before and after

### Before

```yaml
modules:
  - class: modules::api::RestApiModule
    config:
      host: 0.0.0.0
      port: 3111

  - class: modules::stream::StreamModule
    config:
      host: 0.0.0.0
      port: 3112
      adapter:
        class: modules::stream::adapters::RedisAdapter
        config:
          redis_url: redis://localhost:6379

  - class: modules::queue::QueueModule
    config:
      adapter:
        class: modules::queue::BuiltinQueueAdapter

  - class: modules::cron::CronModule
    config:
      adapter:
        class: modules::cron::KvCronAdapter
```

### After

```yaml
workers:
  - name: iii-http
    config:
      host: 0.0.0.0
      port: 3111

  - name: iii-stream
    config:
      host: 0.0.0.0
      port: 3112
      adapter:
        name: redis
        config:
          redis_url: redis://localhost:6379

  - name: iii-queue
    config:
      adapter:
        name: builtin

  - name: iii-cron
    config:
      adapter:
        name: kv
```

## Docker-based workers

A new `image` field lets you declare Docker-based workers:

```yaml
workers:
  - image: docker.io/org/my-worker:latest
    config:
      III_ENGINE_URL: "ws://localhost:49134"
      III_API_URL: "http://localhost:3111"
```

Each entry must have either `name` or `image` (not both).

## Quick search-and-replace

For most configs, you can apply these replacements in order:

1. `modules:` → `workers:`
2. `class: modules::worker::WorkerModule` → `name: iii-worker-manager`
3. `class: modules::stream::StreamModule` → `name: iii-stream`
4. `class: modules::state::StateModule` → `name: iii-state`
5. `class: modules::api::RestApiModule` → `name: iii-http`
6. `class: modules::pubsub::PubSubModule` → `name: iii-pubsub`
7. `class: modules::observability::OtelModule` → `name: iii-observability`
8. `class: modules::queue::QueueModule` → `name: iii-queue`
9. `class: modules::http_functions::HttpFunctionsModule` → `name: iii-http-functions`
10. `class: modules::cron::CronModule` → `name: iii-cron`
11. `class: modules::shell::ExecModule` → `name: iii-exec`
12. `class: modules::bridge_client::BridgeClientModule` → `name: iii-bridge`

Then for adapter entries inside `config:` blocks:

13. `class: modules::stream::adapters::KvStore` → `name: kv`
14. `class: modules::stream::adapters::RedisAdapter` → `name: redis`
15. `class: modules::state::adapters::KvStore` → `name: kv`
16. `class: modules::state::adapters::RedisAdapter` → `name: redis`
17. `class: modules::queue::BuiltinQueueAdapter` → `name: builtin`
18. `class: modules::queue::RedisAdapter` → `name: redis`
19. `class: modules::queue::RabbitMQAdapter` → `name: rabbitmq`
20. `class: modules::cron::KvCronAdapter` → `name: kv`
21. `class: modules::cron::RedisCronAdapter` → `name: redis`
22. `class: modules::pubsub::LocalAdapter` → `name: local`
23. `class: modules::pubsub::RedisAdapter` → `name: redis`

<Callout type="warn">
The old `modules:` key and `class:` field are **no longer accepted**. The engine will fail to start if it encounters the old format.
</Callout>
