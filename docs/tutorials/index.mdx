## Learning paths

Pick the track that matches what you are trying to do. Tracks are
independent — you can do them in any order — but inside a track, the
tutorials build on each other.

### Track 1 — Your first useful backend

For solo developers and early-stage builders. Goal: a real backend running
in under 30 minutes without writing infra glue.

<CardGroup cols={2}>
  <Card title="1. CRUD API in 10 minutes" href="/tutorials/crud-api-in-10-minutes" icon="database">
    Add `todo-worker` and `iii-http`, get a persistent REST API with no
    server code.
  </Card>
  <Card title="2. Add image uploads" href="/tutorials/add-image-uploads" icon="image">
    Compose `image-resize` with the todo app via channels.
  </Card>
  <Card title="3. Scheduled jobs and observability" href="/tutorials/scheduled-jobs-and-observability" icon="clock">
    Cron-trigger a cleanup over state and watch the trace in
    `iii-observability`.
  </Card>
</CardGroup>

### Track 2 — Adopt iii incrementally

For teams with an existing stack. Goal: slot iii in one capability at a
time, without rewriting your services.

<CardGroup cols={2}>
  <Card title="4. Bridge an existing HTTP API" href="/tutorials/bridge-existing-api" icon="bridge">
    Wrap a legacy service with `iii-bridge` and add retry semantics.
  </Card>
  <Card title="5. Background jobs without a job runner" href="/tutorials/background-jobs-without-a-runner" icon="list-check">
    Replace Sidekiq / BullMQ / Celery for the simple case with `iii-queue`.
  </Card>
  <Card title="6. Real-time dashboard" href="/tutorials/realtime-dashboard" icon="chart-line">
    Push live state changes to a browser with `iii-stream` and the browser
    SDK.
  </Card>
</CardGroup>

### Track 3 — iii for AI agents

For agent and AI-tooling builders. Goal: use iii as the substrate for
agents — tools, memory, scheduling, streaming.

<CardGroup cols={2}>
  <Card title="7. Expose your functions as MCP tools" href="/tutorials/expose-functions-as-mcp-tools" icon="plug">
    Add the `mcp` worker and turn every tagged iii function into an agent
    tool.
  </Card>
  <Card title="8. Build a tool-using agent worker" href="/tutorials/build-a-tool-using-agent" icon="robot">
    Register an LLM-backed function that calls other iii functions and
    streams tokens back.
  </Card>
  <Card title="9. AI-driven browser tests" href="/tutorials/ai-driven-browser-tests" icon="flask">
    Add `proof` and let it generate Playwright plans against your app.
  </Card>
  <Card title="10. Durable agent memory" href="/tutorials/durable-agent-memory" icon="brain">
    Combine `iii-state` and `iii-cron` to give an agent persistent memory
    and scheduled reflection.
  </Card>
</CardGroup>
