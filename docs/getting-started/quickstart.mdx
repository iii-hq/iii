## Scaffold the project

Use the iii project template to bootstrap an engine config plus a worker or two.

## Start the engine

Launch the iii engine pointing at the scaffold's `config.yaml`.

## Start a Python worker

Run the Python SDK worker in the scaffold and watch it register its functions with the engine.

## Start a TypeScript worker

Run the Node SDK worker and watch it register its functions alongside the Python worker.

## Call across languages

Trigger a function on one worker from the other to demonstrate cross-language invocation.

## Add state

Wire functions to the iii-state worker to persist key-value data across calls.

## Add HTTP endpoints

Bind functions to HTTP triggers so the scaffold answers requests via iii-http.

## How it works

Recap how the primitives — workers, functions, triggers — fit together at runtime in this scaffold.

## Explore with the console

Open the iii console to inspect connected workers, registered functions, triggers, state, and traces.
