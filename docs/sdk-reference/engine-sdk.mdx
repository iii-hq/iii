## Message types

The set of frame types exchanged over the engine WebSocket.

What a worker sends after the WebSocket opens — registering its functions and triggers.

Sync vs fire-and-forget call paths between client, engine, and worker.

Shape of `invokefunction` frames.

## Invocation result message

Success and error payload shapes for `invocationresult`.

## Engine-collected invocation metrics

Engine-side metrics emitted regardless of SDK language: `iii.invocations.total`,
`iii.invocation.duration`, `iii.invocation.errors.total`.

## Engine discovery functions

Built-in functions and triggers for inspecting and managing system state: `engine::functions::list`,
`engine::workers::list`, `engine::triggers::list`, `engine::trigger-types::list`,
`engine::workers::register`, and the `engine::functions-available` and `engine::workers-available`
triggers.

## Disable telemetry

Two distinct toggles control what iii sends out: `III_TELEMETRY_ENABLED` governs anonymous usage
analytics (the iii-telemetry worker), and `OTEL_ENABLED` governs OpenTelemetry instrumentation (the
iii-observability worker). Set either to `false` to disable that channel independently.
