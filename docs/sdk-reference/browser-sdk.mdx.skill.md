<Note>
  Logging and observability surfaces are documented with the [iii-observability worker](#); that worker owns logging and OpenTelemetry end-to-end. The browser SDK does not currently ship OpenTelemetry.

Channel surfaces (`Channel`, `ChannelReader`, `ChannelWriter`, `ChannelDirection`,
`StreamChannelRef`, `createChannel`) are documented with the [iii-worker-manager](#); channels are
managed by that worker end-to-end.

## Installation

How to add the browser SDK to a web project.

## Initialization

Connecting to the engine and obtaining the SDK handle. The handle is conventionally named `worker`.

## Connection lifecycle

How the SDK connects to the engine, registers functions, and reconnects with re-registration on
disconnect (exponential backoff with jitter, configured via the reconnection options on
`InitOptions`).

## Methods

The functions exposed by the browser SDK for registering, invoking, and tearing down workers.

### registerWorker

Register this worker with an iii engine.

#### InitOptions

Options accepted when constructing the worker instance (worker name, timeouts,
`reconnectionConfig`).

### registerFunction

Register a callable function on this worker, with optional request and response JSON Schema formats.

### registerTrigger

Bind a registered function to a configured trigger instance.

### trigger

Invoke a registered function or fire a trigger from the browser.

### shutdown

Disconnect from the engine and release resources.

### registerTriggerType

Declare a new trigger type that this worker can source events from. **Consider removing**

### unregisterTriggerType
