Workers are modular components that extend the iii engine with specific capabilities. Each worker registers functions and triggers with the engine.

For conceptual background on how workers fit into the iii architecture, see [Primitives: Functions, Triggers, Workers](/primitives-and-concepts/functions-triggers-workers). For engine configuration, see [Engine Configuration Reference](/api-reference/engine-config).

## Architecture

Workers run in three modes:

- **Engine workers** (Rust): Compiled into the engine binary, highest performance
- **SDK workers** (Node/Python/Rust): Connect via WebSocket, register functions
- **Managed workers**: Engine-spawned processes with automatic lifecycle management

Each worker can configure adapters for external backends. For example, the Queue worker can use Redis or RabbitMQ as its message broker.
