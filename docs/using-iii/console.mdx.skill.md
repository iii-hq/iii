# Console


<Note>
  The console is the visual UI for iii systems. It lists workers, functions, and triggers (via the
  [iii-engine-functions worker](#)) and shows traces, logs, and metrics (via the [iii-observability
  worker](#)). For scripting or agent integration, call those workers' functions directly.
</Note>

## Launch the console

Run `iii console` and open `http://127.0.0.1:3113`. The engine must already be running; the console
connects to it.

## Workers page

Lists every worker process currently connected to the engine, with live metrics, runtime and
isolation badges, and the functions each worker has registered.

## Functions page

Lists every registered function with its metadata, and provides a JSON editor to invoke any function
directly with a custom payload.

## Triggers page

Lists every registered trigger grouped by type, with interactive test tools per type (HTTP request
builder, cron "Trigger Now", event emitter, state edit).

## States page

Browser for the engine's key-value state store with full CRUD on entries grouped by scope.

## Streams page

Live WebSocket monitor showing messages flowing through the engine's stream connections.

## Queues page

Lists durable topics with subscriber counts, in-flight, retries, and dead-letter activity, plus a
publisher for sending test messages.

## Traces page

OpenTelemetry trace visualization with waterfall, flame graph, service breakdown, trace map, and
flow views, plus span details and filtering.

## Logs page

Structured OpenTelemetry log viewer with severity filter, full-text search, time-range controls, and
trace correlation.

## Flow page

Opt-in interactive graph of triggers, functions, state stores, and queues. Enable with
`--enable-flow` or `III_ENABLE_FLOW`.

## Configuration

CLI flags, environment variables, and default ports (3111 engine HTTP, 3112 engine WS, 3113 console
UI, 49134 SDK WebSocket).
