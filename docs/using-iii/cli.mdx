## Invoking functions through the cli

`iii trigger --function-id=... --payload=...` invokes a registered function against a running engine; pass `--address` / `--port` to target a remote engine.

## Publishing triggers through the cli

Firing a trigger from the command line so external systems and scripts can publish events into the engine.

## CLI arguments

The flags accepted by the `iii` binary (`--config`, `--use-default-config`, `--version`).

## Managing iii Cloud deployments

`iii cloud` manages hosted iii deployments — see [Deployment → iii Cloud deployments](/using-iii/deployment#iii-cloud-deployments).

## Updating iii itself

`iii update [target]` updates the iii engine and its managed binaries; distinct from `iii worker update`, which updates pinned workers.
