## Initialization

The SDK connects your application to the iii engine over WebSocket.

<Tabs>
<Tab title="Node / TypeScript">
```typescript
import { registerWorker } from 'iii-sdk'

const iii = registerWorker('ws://localhost:49134', {
  workerName: 'my-worker',
  invocationTimeoutMs: 30000,
})
```
</Tab>
<Tab title="Python">
```python
from iii import III

iii = III('ws://localhost:49134')
await iii.connect()
```
</Tab>
<Tab title="Rust">
```rust
use iii_sdk::III;

let iii = III::new("ws://localhost:49134");
iii.connect().await?;
```
</Tab>
</Tabs>

<Info title="Default port">
  The iii engine listens on `ws://localhost:49134` by default when running locally.
</Info>

### InitOptions (TypeScript)

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `workerName` | `string` | `undefined` | Name for this worker instance |
| `enableMetricsReporting` | `boolean` | `true` | Enable OpenTelemetry metrics |
| `invocationTimeoutMs` | `number` | `30000` | Default timeout for `trigger()` calls |
| `reconnectionConfig` | `Partial<ReconnectionConfig>` | See below | Reconnection behavior |
| `otel` | `OtelConfig` | `undefined` | OpenTelemetry configuration |

### ReconnectionConfig

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `initialDelayMs` | `number` | `1000` | Initial reconnection delay |
| `maxDelayMs` | `number` | `30000` | Maximum reconnection delay |
| `backoffMultiplier` | `number` | `2` | Exponential backoff multiplier |
| `jitterFactor` | `number` | `0.3` | Jitter factor for delay randomization |
| `maxRetries` | `number` | `-1` | Maximum retries (`-1` for unlimited) |

## trigger()

Invokes a function using a request object. Behavior depends on the optional `action`:

| Action | Behavior | Returns |
|--------|----------|---------|
| (none) | Synchronous invocation — waits for the function result | `Promise<TOutput>` |
| `TriggerAction.Enqueue({ queue })` | Enqueues work to a named queue; waits for engine acknowledgement | `Promise<EnqueueResult>` |
| `TriggerAction.Void()` | Fire-and-forget — no response awaited | `Promise<void>` / `None` |

<Tabs>
<Tab title="Node / TypeScript">
```typescript
import { registerWorker, TriggerAction } from 'iii-sdk'

const iii = registerWorker('ws://localhost:49134')

const result = await iii.trigger<{ name: string }, { id: string }>({
  function_id: 'users::create',
  payload: { name: 'Alice' },
})

await iii.trigger({
  function_id: 'orders::process',
  payload: { orderId: '123' },
  action: TriggerAction.Enqueue({ queue: 'payment' }),
})

iii.trigger({
  function_id: 'notifications::send',
  payload: { to: 'alice@example.com' },
  action: TriggerAction.Void(),
})
```
</Tab>
<Tab title="Python">
```python
from iii import III, TriggerAction

iii = III('ws://localhost:49134')
await iii.connect()

result = await iii.trigger({
    'function_id': 'users::create',
    'payload': {'name': 'Alice'},
})

await iii.trigger({
    'function_id': 'orders::process',
    'payload': {'orderId': '123'},
    'action': TriggerAction.Enqueue(queue='payment'),
})

await iii.trigger({
    'function_id': 'notifications::send',
    'payload': {'to': 'alice@example.com'},
    'action': TriggerAction.Void(),
})
```
</Tab>
<Tab title="Rust">
```rust
use iii_sdk::{III, TriggerRequest, TriggerAction};
use serde_json::json;

let iii = III::new("ws://localhost:49134");
iii.connect().await?;

let result = iii.trigger(
    TriggerRequest::new("users::create", json!({"name": "Alice"}))
).await?;

iii.trigger(
    TriggerRequest::new("orders::process", json!({"orderId": "123"}))
        .action(TriggerAction::enqueue("payment")),
).await?;

iii.trigger(
    TriggerRequest::new("notifications::send", json!({"to": "alice@example.com"}))
        .action(TriggerAction::void()),
).await?;
```
</Tab>
</Tabs>

<ResponseField name="trigger" type="<TInput, TOutput>(request: TriggerRequest&lt;TInput&gt;) => Promise&lt;TOutput&gt;" required>
  Triggers a registered function. Behavior depends on the optional `action` field.

  <AccordionGroup>
    <Accordion iconName="Settings" title="TriggerRequest fields" description="">
      <ResponseField name="function_id" type="string" required>
        The function identifier (e.g., `'users::create'`, `'state::set'`).
      </ResponseField>
      <ResponseField name="payload" type="TInput" required>
        The payload to pass to the function.
      </ResponseField>
      <ResponseField name="action" type="TriggerAction">
        Optional action: `TriggerAction.Enqueue({ queue })` for named queues, `TriggerAction.Void()` for fire-and-forget. Omit for synchronous invocation.
      </ResponseField>
      <ResponseField name="timeout_ms" type="number">

      </ResponseField>
    </Accordion>
    <Accordion iconName="ArrowUpRight" title="Returns" description="">
      <ResponseField name="result" type="Promise&lt;TOutput&gt;" required>

      </ResponseField>
    </Accordion>
  </AccordionGroup>
</ResponseField>

## registerFunction()

Registers a function with the iii engine. Once registered, the function can be triggered by any other component in the system.

<Tabs>
<Tab title="Node / TypeScript">
```typescript
const ref = iii.registerFunction(
  { id: 'users::create' },
  async (data: { name: string }) => {
    return { id: crypto.randomUUID(), name: data.name }
  }
)
```
</Tab>
<Tab title="Python">
```python
async def create_user(data):
    return {'id': str(uuid.uuid4()), 'name': data['name']}

ref = iii.register_function('users::create', create_user)
```
</Tab>
<Tab title="Rust">
```rust
iii.register_function("users::create", |data: Value| async move {
    Ok(json!({"id": Uuid::new_v4().to_string(), "name": data["name"]}))
});
```
</Tab>
</Tabs>

## registerTriggerType()

Registers a custom trigger type with the engine. This allows you to define new ways to invoke functions beyond the built-in types (HTTP, cron, queue).

<Tabs>
<Tab title="Node / TypeScript">
```typescript
iii.registerTriggerType<WebhookConfig>(
  { id: 'webhook', description: 'External webhook trigger' },
  {
    async registerTrigger(config) {
      // set up the webhook listener
    },
    async unregisterTrigger(config) {
      // tear down the webhook listener
    },
  }
)
```
</Tab>
<Tab title="Python">
```python
iii.register_trigger_type(
    id='webhook',
    description='External webhook trigger',
    handler=WebhookHandler(),
)
```
</Tab>
<Tab title="Rust">
```rust
iii.register_trigger_type("webhook", "External webhook trigger", WebhookHandler);
```
</Tab>
</Tabs>

## Event Callbacks

### onFunctionsAvailable()

Receives the current function list whenever the engine announces changes.

```typescript
const unsubscribe = iii.onFunctionsAvailable((functions) => {
  console.log('Available functions:', functions.map(f => f.id))
})
```

### onLog()

Receives OpenTelemetry log events from the engine. Useful for building custom log UIs or forwarding logs.

```typescript
const unsubscribe = iii.onLog(
  (log) => console.log(`[${log.severity_text}] ${log.body}`),
  { level: 'info' }
)
```

| Level | Description |
|-------|-------------|
| `'all'` | All log events (default) |
| `'trace'` | Trace and above |
| `'debug'` | Debug and above |
| `'info'` | Info and above |
| `'warn'` | Warn and above |
| `'error'` | Error and above |
| `'fatal'` | Fatal only |

## Connection State

Monitor and react to connection state changes.

```typescript
const state = iii.getConnectionState()

const unsubscribe = iii.onConnectionStateChange((state) => {
  console.log('Connection:', state)
})
```

| State | Description |
|-------|-------------|
| `'disconnected'` | Not connected |
| `'connecting'` | Initial connection in progress |
| `'connected'` | Connected and ready |
| `'reconnecting'` | Lost connection, attempting to reconnect |
| `'failed'` | Reconnection attempts exhausted |

## Types

### TriggerRequest

Request object for `trigger()`. All SDKs use this shape.

```typescript
type TriggerRequest<TInput = unknown> = {
  function_id: string
  payload: TInput
  action?: TriggerAction
  timeoutMs?: number   // Node (camelCase convention)
  timeout_ms?: number  // Python / Rust (snake_case convention)
}
```

All SDKs use milliseconds with a default of 30000.

### TriggerAction

Controls how the engine routes the invocation: synchronous, enqueue to a named queue, or fire-and-forget.

```typescript
type TriggerAction =
  | { type: 'enqueue'; queue: string }
  | { type: 'void' }
```

| Action | Use |
|--------|-----|
| `TriggerAction.Enqueue({ queue })` | Enqueue work to a named queue |
| `TriggerAction.Void()` | Fire-and-forget, no response |

### EnqueueResult

Returned when `trigger()` is called with `action: TriggerAction.Enqueue({ queue })`.

```typescript
type EnqueueResult = {
  messageReceiptId: string
}
```

### ApiRequest

Received by functions registered as HTTP trigger handlers.

```typescript
type ApiRequest<TBody = unknown> = {
  path_params: Record<string, string>
  query_params: Record<string, string | string[]>
  body: TBody
  headers: Record<string, string | string[]>
  method: string
}
```

### ApiResponse

Returned by HTTP trigger handler functions.

```typescript
type ApiResponse<TStatus extends number = number, TBody = string | Buffer | Record<string, unknown>> = {
  status_code: TStatus
  headers?: Record<string, string>
  body: TBody
}
```

## Sub-modules

The SDK exports the following from the main package:

| Export | Description |
|--------|-------------|
| `registerWorker` | Create and connect an SDK instance (Node); Python/Rust use `III` + `connect()` or `register_worker()` |
| `TriggerAction` | Factory for `Enqueue` and `Void` actions |
| `TriggerRequest` | Request type for `trigger()` |
| `EnqueueResult` | Result type when enqueueing to a named queue |

Additional modules for specific functionality:

| Import | Description | Docs |
|--------|-------------|------|
| `iii-sdk/stream` | Stream data types and interfaces | [Stream module](/modules/module-stream) |
| `iii-sdk/state` | State management types and interfaces | [State module](/modules/module-state) |
| `iii-sdk/telemetry` | OpenTelemetry utilities | [Observability module](/modules/module-observability) |
