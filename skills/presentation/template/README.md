# __SPEC_NAME__ — presentation

An interactive, build-in-public presentation of the **__SPEC_NAME__** tech
spec. Generated by the `/presentation` skill from the spec in this repo.

A standalone Vite + React + TypeScript site. No backend. The architecture is a
navigable map, the flows are steppable, and the whole thing argues *why* the
work matters.

## Develop

```bash
pnpm install --ignore-workspace
pnpm dev          # http://localhost:5173
```

## Build & share

```bash
pnpm build        # -> dist/  (static, relative paths)
pnpm preview      # sanity-check the production build
```

`dist/` is a fully portable static site (`base: './'`). Drop it on any CDN,
GitHub Pages, S3, or open `dist/index.html` straight from disk. Nothing to
configure, safe to share publicly.

```bash
pnpm typecheck    # strict TypeScript, no emit
```

## Structure

- `src/content/deck.ts` — nav, wordmark, footer (the deck's identity).
- `src/content/*.ts` — the data each slide renders from.
- `src/sections/*.tsx` — one thin section per slide.
- `src/pages/*.tsx` — deep-dive pages (`#/<slug>` routes).
- `src/components/` — the frozen design system: primitives, the interactive
  diagram archetypes, hooks, and the layout shell. The look and feel live here.

Edit content in `content/` + `sections/`; leave `components/` alone unless you
are extending the system itself.
